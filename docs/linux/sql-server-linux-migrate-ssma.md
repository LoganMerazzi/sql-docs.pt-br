# <a name="automate-database-migration-to-linux-with-the-sql-server-migration-assistant"></a>Automatizar a migração de banco de dados para Linux com o SQL Server Migration Assistant

[!INCLUDE[appliesto-ss-xxxx-xxxx-xxx-md-linuxonly](../includes/appliesto-ss-xxxx-xxxx-xxx-md-linuxonly.md)]

Este artigo apresenta [SQL Server Migration Assistant (SSMA)](https://msdn.microsoft.com/library/mt613434.aspx) que ajuda você a migrar facilmente os bancos de dados para o SQL Server no Linux do Microsoft Access, DB2, MySQL, Oracle e Sybase. O SSMA é um aplicativo do Windows, portanto, use o SSMA quando você tiver um computador Windows que pode se conectar a uma instância remota do SQL Server no Linux. 

O SSMA dá suporte a uma variedade de bancos de dados de origem incluindo Oracle, MySQL, Sybase, DB2 e Microsoft Access para o SQL Server no Linux e ajuda a automatizar tarefas de migração, como:

- Avalie seu banco de dados de origem
- Converter o esquema de banco de dados de origem ao esquema do Microsoft SQL Server
- Migrar o esquema
- Migrar os dados
- Testar a migração

Para começar, baixe o SQL Server Migration Assistant (SSMA) do banco de dados de origem na lista a seguir:
- [SSMA para Access](https://aka.ms/ssmaforaccess)
- [SSMA para DB2](https://aka.ms/ssmafordb2)
- [SSMA para MySql](https://aka.ms/ssmaformysql) 
- [SSMA para Oracle](https://aka.ms/ssmafororacle)
- [SSMA para Sybase ASE](https://aka.ms/ssmaforsybase) 

Em seguida, siga as [SQL Server Migration Assistant (SSMA)](https://msdn.microsoft.com/library/mt613434.aspx) migrar seu banco de dados de origem para o SQL Server no Linux.
